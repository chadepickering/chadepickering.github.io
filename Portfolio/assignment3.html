import pandas as pd
import os
import os.path

# Path to files
prefix = "assignment3_data/"
food_type = prefix + "fruit/" # full, will use in for loop later
files_dir = os.listdir(food_type)

# Use list comprehension to get list of .xlsx files
files = [x for x in files_dir if x.endswith("xlsx")]

# Empty df
data_df = pd.DataFrame()

# Create beginning of 2 columns for type (fruit, etc.) and product 
type_product = [['type','food']]

for h in range(len(files)):    
    # Iterate through paths to each fruit
    product_i = food_type + files[h]
    
    # Removes all cells with NaN values
    prod_remNA = pd.read_excel(product_i, header = 1).dropna()
    
    # Drops the columns with units
    prod_fresh = prod_remNA.drop(prod_remNA.columns[[2, 5]], axis = 1)
    
    # Finds desired row that begins with "Fresh"; iloc[:,0] identifies look in the first col
    prod_fresh = prod_fresh[prod_fresh.iloc[:,0].str.endswith('Fresh1')] 
            
    # Appends new "fresh" row to the bottom of the growing df
    data_df = data_df.append(prod_fresh, ignore_index = True)[prod_fresh.columns.tolist()]
   
    # If df is not empty (some fruits have no "fresh" row), 
    # get type (in this case fruit) and product name for the "type_product"
    if not prod_fresh.empty:
        prod_name = ['fruit', files[h][:-5]]
        type_product.append(prod_name)

# Create df with left-most columns for final df, "type" and "food"
food_df = pd.DataFrame(type_product[1:], columns = type_product[0])

# Removes all columns with any NA values if they exist
data_df = data_df.dropna(1)

# Concatenate food and data dfs column-wise 
fruit_df = pd.concat([food_df, data_df], axis = 1, join='inner')

# Use dictionary-like rename function to make the column names match the example
fruit_df = fruit_df.rename(index = str, 
                               columns = {"type":"type",
                                        "food":"food",
                                        "Form":"form",
                                        "Average retail price ":"price_per_lb",
                                        "Preparation":"yield",
                                        "Size of a ":"lb_per_cup",
                                        "Average price":"price_per_cup"})

Does your code produce the correct prices for tomatoes? If not, why not? Do any other files have the same problem as the tomatoes file? Because "Fresh" is a category containing multiple kinds of tomatoes and not a single row as expected, no one price (or any price in my case) is returned. There are multiple accurate prices depending on type of tomato, so this format is not compatible here. Other files such as "broccoli," "carrots," "cauliflower," "celery," "lettuce_romaine," "mushrooms," and "spinach" have the same issue, where there are multiple kinds of the product, and therefore multiple "Fresh" prices. There are also vegetables such as "cabbage" that matched with "Fresh1" but had valid entries for a subgroup just after it in the row as usual. This created problems because as the iteration continued, the next "Fresh1" matches in "cabbage", for example, would input the data in the "collard greens" row.
